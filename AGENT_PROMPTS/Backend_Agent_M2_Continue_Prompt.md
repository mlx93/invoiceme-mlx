# Backend Agent M2 - Continue Implementation

Continue M2 implementation by implementing Customer CRUD vertical slices following CQRS and Vertical Slice Architecture: create `customers/createcustomer/` (CreateCustomerCommand, CreateCustomerHandler with @Transactional, CreateCustomerValidator, CreateCustomerController), `customers/getcustomer/` (GetCustomerQuery, GetCustomerHandler, GetCustomerController), `customers/listcustomers/` (ListCustomersQuery with filters/pagination, ListCustomersHandler returning Page<CustomerSummaryResponse>, ListCustomersController), `customers/updatecustomer/` (UpdateCustomerCommand, UpdateCustomerHandler, UpdateCustomerValidator, UpdateCustomerController), and `customers/deletecustomer/` (DeleteCustomerCommand, DeleteCustomerHandler, DeleteCustomerController). Each vertical slice should include MapStruct mappers (Command → Entity, Entity → Response DTO), use existing CustomerRepository methods, publish domain events via DomainEventPublisher after transaction commit, and follow the patterns documented in `/backend/M2_IMPLEMENTATION_PROGRESS.md`. After Customer CRUD is complete and tested, proceed to Invoice CRUD vertical slices (create, get, list, update, markAsSent, cancel) and then Payment CRUD (recordPayment, get, list) to complete the core E2E flow.
